# pbx-install-recover — resume a partial install

This skill recovers from interrupted PBX Stratos installs. The `pbx-install` wizard has multiple steps (deps → pm2 → scheduled tasks → dashboard → wallet → live trading); if any step fails, this skill detects what's done and resumes from the first incomplete step.

## When to invoke

- User says "install crashed", "install died", "install hung at step X", "resume install"
- User ran `install.bat` / `install.ps1` / `install.sh` and got a partial result
- After a system event (power loss, kill -9, etc.) interrupted an install
- After a fresh git clone where the user wants to verify install state

## Skip when

- No `package.json` exists (not even a clone — start with `pbx-install` from scratch)
- Install completed successfully and user just wants to upgrade (use `pbx-upgrade` instead)
- User wants to wipe and re-install fresh (`uninstall.bat` then `pbx-install`)

## Step 1 — Detect install state checkpoint-by-checkpoint

Run this detection in order. Each checkpoint is independent; record `done` / `missing` / `unknown` / `degraded` for each.

```bash
# Checkpoint 1: .tooling directory + ready marker
ls .tooling/ 2>/dev/null
cat .tooling/ready.json 2>/dev/null | head -5

# Checkpoint 2: npm deps
ls node_modules/.package-lock.json 2>/dev/null  # presence indicates npm install ran

# Checkpoint 3: Python venv
ls .venv/Scripts/python.exe 2>/dev/null         # Windows
ls .venv/bin/python 2>/dev/null                 # macOS/Linux

# Checkpoint 4: pbx_trader_lab installed in venv
.venv/Scripts/python.exe -c "import pbx_trader_lab" 2>&1 | head -5  # Windows
.venv/bin/python -c "import pbx_trader_lab" 2>&1 | head -5          # macOS/Linux

# Checkpoint 5: pm2 installed globally
which pm2 2>/dev/null
pm2 --version 2>/dev/null

# Checkpoint 6: pm2 apps registered
pm2 jlist 2>/dev/null | grep -E "bear-watch-server-stratos|paper-trade-bot-stratos"

# Checkpoint 7: pm2 apps online
pm2 list 2>/dev/null | grep -E "bear-watch-server-stratos|paper-trade-bot-stratos" | grep online

# Checkpoint 8: runtime/bots/local.env present (auto-generated by server first boot)
ls runtime/bots/local.env 2>/dev/null

# Checkpoint 9: scheduled tasks registered (Windows only)
schtasks /query /fo csv 2>/dev/null | grep STRATOS

# Checkpoint 10: server reachable
curl -s -m 5 http://localhost:8787/health

# Checkpoint 11: runtime/lab/user-profile.json present (created by install wizard's personality quiz)
cat runtime/lab/user-profile.json 2>/dev/null | head -5
```

## Step 2 — Produce a state report

```markdown
## Install state detected

| # | Checkpoint | Status | Notes |
|---|---|---|---|
| 1 | .tooling/ + ready.json | ✓ done / ✗ missing | <if missing: bootstrap.ps1/sh hasn't run> |
| 2 | npm deps | ✓ done / ✗ missing | |
| 3 | Python venv | ✓ done / ✗ missing | |
| 4 | pbx_trader_lab installed | ✓ done / ✗ missing | |
| 5 | pm2 installed | ✓ done / ✗ missing | |
| 6 | pm2 apps registered | ✓ done / ✗ missing | |
| 7 | pm2 apps online | ✓ online / ✗ stopped | |
| 8 | local.env present | ✓ done / ✗ missing | |
| 9 | scheduled tasks registered | ✓ N tasks / ✗ none / ⚠ partial (M of N) | (Windows only) |
| 10 | server reachable | ✓ /health ok / ✗ down | |
| 11 | user-profile.json | ✓ done / ✗ missing | <if missing: personality quiz hasn't run> |

**First incomplete step:** <step name>
**Resume action:** <what command to run>
```

## Step 3 — Ask user consent for resume

Don't auto-resume. Show the report. Ask:

> "Detected your install is incomplete at step <N: name>. The recommended resume action is `<command>`. Proceed?"

User can also choose to:
- Skip a step (e.g., they don't want pm2 — they'd run with `npm run server` directly)
- Re-run from an earlier step (e.g., they want to wipe node_modules and redo)
- Stop and investigate manually

## Step 4 — Resume action per checkpoint

| If first failed checkpoint is... | Resume action |
|---|---|
| 1 (.tooling/) | Start-Process install.bat (Windows) or `bash install.sh` (macOS/Linux). The installer's Step 1 reaches bootstrap.ps1/sh and writes ready.json. |
| 2 (npm deps) | `npm install` at repo root |
| 3 (Python venv) | `python -m venv .venv && .venv/Scripts/pip install -e .` (Windows) or `python3 -m venv .venv && .venv/bin/pip install -e .` |
| 4 (pbx_trader_lab) | `.venv/Scripts/pip install -e .` (Windows) or `.venv/bin/pip install -e .` |
| 5 (pm2 not installed) | `npm install -g pm2` |
| 6 (pm2 apps not registered) | `pm2 start bear-watch/pm2.config.cjs && pm2 save` |
| 7 (pm2 apps stopped) | `pm2 restart bear-watch-server-stratos paper-trade-bot-stratos` |
| 8 (local.env missing) | Restart server — it auto-generates on first boot (`pm2 restart bear-watch-server-stratos`) |
| 9 (scheduled tasks missing) | `powershell -ExecutionPolicy Bypass -File bear-watch/register-scheduled-tasks.ps1` (Windows) — invoked via install.bat to avoid the Auto-mode classifier |
| 10 (server down) | Check pm2 logs first: `pm2 logs bear-watch-server-stratos --err --lines 50` |
| 11 (user-profile.json missing) | Invoke `pbx-personality-quiz` skill |

## Step 5 — After each resume, re-check the same checkpoint

Don't auto-advance to the next step. After running the resume action:

1. Re-run the specific checkpoint check
2. Confirm it's now `done`
3. Report success
4. Ask if user wants to continue to the next incomplete step

This lets the user catch unexpected issues per-step rather than chaining failures.

## Step 6 — Idempotent re-invoke

Safe to re-invoke this skill at any time. If everything's done, the report will show all checkpoints as ✓ and the skill exits without asking for any resume.

## Special situations

### User opted out of live trading

If `runtime/lab/user-profile.json` shows `goal: "paper-only"` or `consent_level: "explore-only"`, do NOT prompt for:
- `HELIUS_MAINNET_URL` env var setup
- HD wallet generation
- Live bot fleet enablement

The install is "complete" for the user's chosen scope even if those steps weren't done.

### User opted out of scheduled tasks

If `runtime/lab/user-profile.json` notes scheduled-task opt-out, do NOT count checkpoint 9 as a failure. Mark as "skipped by user choice".

### Windows MSIX sandbox issue

If pm2 isn't found in PATH but the user is running this from Claude Desktop's terminal, surface the known issue: "Claude Desktop runs in MSIX sandbox; pm2 installed there isn't visible to system processes (scheduled tasks). Solution: reinstall pm2 from a real PowerShell window."

## What NOT to do

- DO NOT auto-resume without explicit user consent — always ask.
- DO NOT skip ahead to subsequent steps if an earlier one is failing — fix one at a time.
- DO NOT re-run completed steps "just to be safe" — wastes time + risks regression.
- DO NOT prompt for HD wallet / live trading setup if user opted for paper-only mode.
- DO NOT touch `runtime/bots/local.env` directly — let the server auto-generate on first boot.
- DO NOT modify `.env` without explicit user consent (Tier 3 file).
- DO NOT restart the bot server if it's running and has an open position without explicit user OK (Tier 1+ rule).
