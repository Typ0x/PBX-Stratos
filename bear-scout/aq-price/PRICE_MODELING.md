# AQ → Price modelling  A modelling track in **PBX Stratos**: **predict PBX region-token PRICE moves from air quality.** Same loop discipline as the backtest factory and the `pbx-air-quality-models` PM2.5 work — frozen harness → experiment log → leaderboard.  ## Why it lives here  This is **trading research**, so it belongs in the trading repo, alongside the backtest factory (`bear-scout/backtest/factory/`) that consumes price signals. The air-quality data is a *data dependency* — read from a `pbx-air-quality-dataset` clone (`$PBX_AQ_DATASET`) — not a reason to co-locate the code. (`pbx-air-quality-models` is, by that project's standing rule, air-quality-only and rejects price/trading PRs.)  The model maps **AQ across all three regions → price**: the PBX rebalance engine sets each region's target allocation ∝ 1/(PM2.5·price), so air quality and price are linked by a known protocol formula — see the rebalance-target section below.  ## Files  | File | Role | |---|---| | `price_data.py` | Fetch PBX token prices (GeckoTerminal), clean outliers, join to the AQ hourly tables → `data/price_aq.parquet` | | `rebalance_target.py` | Faithful port of the on-chain rebalance engine — target ∝ efficiency, AMM price impact | | `price_harness.py` | **Frozen** AQ→price evaluation — five framings, persistence baseline | | `price_loop.py` | One experiment cycle → `price_experiments.jsonl` | | `price_leaderboard.py` | Regenerate `PRICE_LEADERBOARD.md` | | `price_test.py` | Tests — ladder, embargo, no-lookahead, rebalance formula | | `price_backtest.py` | Cost-aware walk-forward backtest of the AQ signal vs buy-and-hold | | `run_price_loop.sh` | Sequential self-check loop (local commits, no push, no PRs) |  ## The three framings  Plain price regression at short horizons ~ties persistence (the same lesson the PM2.5 work found). The signal is in the **category**:  - **`rise`** — a classifier for P(price rises ≥ `thr`). The probability   is the confidence: `price_harness` sweeps the decision threshold so the   signal can be dialled to a chosen precision/recall point. - **`bucket`** — a 5-class return classifier (big-down…big-up). The class   probabilities also give an expected return: Σ P(bucket)·(train-set   bucket mean). - **`return`** — plain regression, kept as the honest negative control.  ## Regime keying  A +1 µg/m³ PM2.5 move means something very different at 0.5 than at 9. `price_harness` can restrict a model to rows whose **current PM2.5** is in a band (`pm25_lo/pm25_hi`), so each regime gets its own expert.  ## Rebalance-target model (the protocol-grounded AQ→price link)  `rebalance_target.py` is a **faithful port of the on-chain PBX rebalance engine** (`pbx-platform` / `pbx_solana`, the `YieldMaximizer` strategy) — ported from `constants/mod.rs`, `utils.rs`, `strategies/yield_maximizer/calculations.rs`, `strategies/shared/rebalance_helpers.rs`, and `instructions/whirlpool/math.rs`.  How the engine actually works:  - `pm25_weight` = `WEIGHT_PRECISION² / PM2.5` — a linear inverse, floored   at 1.5 µg/m³. This is only an **input**. - **`efficiency` = weight / price = 1/(PM2.5·price)** — and the engine's   **target allocation is proportional to efficiency**, *not* to 1/PM2.5.   A region that is already expensive gets a *smaller* target — the price   term is the mean-reverting force. `target_allocations` is the accurate   "is it balanced" function and needs PM2.5 **and** price. - It targets an allocation **value share**, trades only when the gap   exceeds a **0.15% dead zone**, clamps the move to **2.5%/cycle**, and a   trade moves price by the **constant-product AMM** formula   `new = old·((R±T)/R)²` (`predict_post_trade_price`).  An earlier version of this model used `1/PM2.5` as the target — wrong, it omits the price term. `evaluate_target_reversion`, `evaluate_bucketed_compose` and `price_backtest.py` have all been **re-run against the faithful `target_allocations`** (the numbers below and in the backtest section are the faithful-target results). `target_weights` remains only as a deprecated price-free shim.  **The faithful re-run did not change the verdict — if anything the signal is slightly weaker.** `target_reversion`'s deviation correlation fell from ~−0.3 (proxy) to roughly −0.2/−0.16/+0.04 across 6/12/24h; `bucketed_compose`'s directional accuracy went 65/66/75% → 62/57/72%. Same shape, marginally attenuated — the price term in the faithful target damps the AQ-driven shift. Conclusion unchanged: a real but weak directional signal, no classifier lift, no tradeable edge.  ## Bucketed-compose — the full AQ-forecast → price chain  `evaluate_bucketed_compose` connects the three pieces end to end:  1. **Bucketed PM2.5 forecast** — the windowed relative-band model    predicts each region's future PM2.5 (expected value from the class    probabilities). 2. **Rebalance formula** — the three forecasts go through    the faithful `target_allocations` → each region's *predicted future*    target share (forecast PM2.5, current price held fixed). 3. **Price signal** — the predicted target-share *change* `dw` (forecast    − current): a region whose air quality is forecast to improve relative    to the others should see its target allocation, hence price, rise.  Result (faithful target): `dw` is a **directional signal** — it calls the forward price direction 62% (6h) / 57% (12h) / 72% (24h), positive correlation (+0.11 to +0.35). Slightly weaker than the earlier 1/PM2.5 proxy (65/66/75%); same shape.  Honest caveats: (1) added as features to the already 32-feature classifier, the composed features give **no macro-F1 lift** (the gradient-boosted model already extracts equivalent signal from the raw features); the compose-only model scores ~0.46–0.49 (above persistence ~0.33, below the full model). The value here is the interpretable directional signal, not a classifier bump. (2) The directional figure is measured only on test rows where all three regions are priced (~100 rows — reported as `signal_n`); treat it as directional, not precise.  ## Perfect-foresight ceiling (diagnostic, NOT a strategy)  `evaluate_aq_price_ceiling` answers "is the weak link the AQ *forecaster* or the AQ→*price mapping*?" by deliberately using the **actual** future PM2.5 (known only in hindsight — not tradeable) in place of a forecast, and comparing:  | horizon | forecast (real) | oracle (perfect future AQ) | gap | |---|--:|--:|--:| | 6h  | 65% dir | 73% | +8 pts | | 12h | 66% dir | 69% | +3 pts | | 24h | 75% dir | 71% | −4 pts |  **Finding: the bottleneck is the AQ→price *relationship*, not the forecaster.** Even with perfect future-AQ knowledge, price direction is only callable ~70% — it does not approach certainty. The gap between the real forecast and the oracle is tiny (+3..+8 pts, negative at 24h within noise), so a better PM2.5 forecaster would buy almost nothing. The AQ→price mapping has a low ceiling. (Samples are ~100–120 rows — the *relative* forecast-vs-oracle finding is robust; the absolute ~70% is itself uncertain.)  ## No-lookahead embargo  Forward-return targets are computed per city before the chronological 80/20 split. Because the pooled frame interleaves three cities, a row-count embargo would be far too small — so the harness embargoes by **time**: a train row qualifies only if its datetime is at least `horizon` hours before the test split. A train row's label can never reach into the test period, on any framing. `price_test.py` asserts this directly on the 3-city interleaved frame.  ## Cross-region  `price_data.py` adds `pm25_vs_others` / `pm25_other_mean` / `price_other_mean` — each region's air quality *relative to the other two*, because the PBX market is relative. Ablation confirms it helps.  ## Run summary (2026-05-19, ~40-day joined window, leakage-free)  31 experiments. **27/27 classification experiments beat the persistence baseline**; regression did not (~ties persistence). Best `rise`: `rise-h6-TOR` (macro-F1 ~0.69). Cross-region AQ helps; price lags help. Hand-picked PM2.5 regime bands were too thin — fixed by re-keying on PM2.5 *quintiles* (`rise-h6-q4` is among the strongest). The two-stage AQ-forecast compose is a negative result. See `PRICE_LEADERBOARD.md` for the live ranking.  ## Cost-aware backtest — inconclusive, no demonstrated edge  `price_backtest.py` turns the AQ signal into an actual strategy (`aq-rotate`: hold the region with the largest positive predicted target-allocation change, else USDC) and replays it walk-forward, **net of 80 bps/leg costs, vs equal-weight buy-and-hold**, 3 folds of 95 bars.  Result against the **faithful `target_allocations`** (vs buy-and-hold, per fold):  | horizon | fold 1 | fold 2 | fold 3 | mean | beat | |---|--:|--:|--:|--:|:-:| | 6h  | −197 | −30 | +311 | +28 pp | 1/3 | | 12h | +51 | +15 | +3 | +23 pp | 3/3 | | 24h | −197 | −40 | +150 | −29 pp | 1/3 |  **5 of 9 fold-results beat buy-and-hold — noise-dominated, not a demonstrated edge.** Fold-to-fold swings are ±200–310 pp (the tokens do 3–4× in a 95-hour fold — volatility swamps any signal). Per-horizon means are inconsistent: h6 +28, h12 +23, h24 *loses* −29 pp. h12's 3/3 sweep is exactly the kind of result that turns up by chance when three horizons are tested — a multiple-comparisons artifact. The strategy also churns 40–90 trades per 95-bar fold. (The earlier 1/PM2.5-proxy run gave 6/9 — the faithful target neither rescued nor changed the verdict.)  Honest verdict: with ~40 days of data, 3 folds, and microcap volatility this severe, the backtest can neither confirm nor rule out a modest edge — it is **inconclusive, leaning weak-at-best**. Combined with the perfect-foresight ceiling (~70% even with a *perfect* AQ forecast), the weight of evidence is that **air quality is a real but weak directional signal, and not a demonstrated money-maker on this market/data.**  ## Honest limitations  - **~40 days only** — GeckoTerminal's free tier retains ~40 days of   hourly bars, and the PBX tokens are young. Small dataset; treat lifts   as directional, not precise. - **No meteorology** — the HRRR backfill has not reached this recent   window, so `met_*` is excluded for now. Re-add once it catches up. - This predicts a price *move category*, not a trading P&L. Turning a   signal into a strategy and backtesting it net-of-cost is the   **PBX Stratos** backtest factory's job — a deliberate, separate step.  ## Two-stage compose — tested, negative result  `evaluate_two_stage` feeds a stage-1 PM2.5 *forecast* into the price model as an extra feature. It does **not** help: macro-F1 0.602 vs a present-AQ control of 0.649 at 6h (−0.048). The price model already extracts what it needs from present air quality; a noisy forecast of future AQ adds noise, not signal. Honest negative result — kept logged.  ## Tests  `python3 price_test.py` — band ladder, 5-band window bounds, the no-lookahead forward-return target, the chronological split, and outlier rejection.  ## Next  - Re-key regimes on PM2.5 level once more price history accumulates. - Re-add MET features once the HRRR backfill reaches the price window.
