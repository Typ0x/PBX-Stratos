# lab — Research scope (BEAR-SCOUT's home + the wallet decoder framework)  This folder owns research and strategy work for a PBX Stratos install: backtest harnesses, paper trading, the strategy registry, wallet decoders, signal investigations, model fitting, near-term PM2.5 → price modeling. Anything that's about DISCOVERING what works (rather than KEEPING it running) lives here.  This is BEAR-SCOUT's working directory.  ## The research-to-live pipeline  ```   IDEA    ↓    ↓ (hypothesis: "I think X correlates with Y in regime Z")    ↓   DECODE         ←  bear-scout/runners/wallet-decoder.py + wallet-evolve.py    ↓                +  agentic-decode.py  (Claude in a loop with DSL evaluator)    ↓    ↓ (decoded rule has positive held-out P&L on walk-forward 70/30 split)    ↓   BACKTEST       ←  bear-scout/runners/  (run against historical data)    ↓    ↓ (if backtest looks promising, deploy to paper)    ↓   PAPER TRADE    ←  bear-scout/runners/paper-trade.py  (live prices, simulated fills)    ↓    ↓ (if paper survives N days and beats the persistence baseline)    ↓   LIVE           ←  bots/src/strategies/  (real money, with consent gates)    ↓                +  packages/swap-router/  (best-venue execution)    ↓    ↓ (monitor weekly, demote if win-rate degrades)    ↓   RETIRE / ITERATE ```  Each step is more expensive than the last. Burn your effort on the cheap steps (idea, decode, backtest) before committing to the expensive ones (paper, live).  ## What's in this folder  | Path | What it does | |------|--------------| | `runners/` | All the runnable research scripts (see decoder table below). | | `runners/paper-trade.py` | The paper trader. Pull features for a pubkey (needs DATABASE_URL or Helius access) python3 bear-scout/runners/wallet-decoder.py <pubkey>  # 2. Systematic decode — evolve hypotheses (default: 10 epochs) python3 bear-scout/runners/wallet-evolve.py <pubkey> --epochs 10  # 3. Train ML model on the decoded snapshots python3 bear-scout/runners/wallet-ml.py <pubkey>  # 4. Iterate with Claude in the loop (refines entry+exit DSL predicates) python3 bear-scout/runners/agentic-decode.py <pubkey> --rounds 10 ```  ## Using the Claude Code skill  `.claude/skills/wallet-decoder/SKILL.md` activates when you ask Claude to "decode this wallet" / "analyze this trader" / paste a pubkey. It knows the prerequisites, the command order (decoder → evolve → ml → agentic-decode), and how to report results in your personality's voice.  ## Research discipline  - **Always have a control.** When evaluating a new strategy, compare   against the persistence baseline ("do nothing for the next N hours")   AND against your current best strategy. Many "winners" only beat   zero — not always meaningful. - **Always document your hypothesis BEFORE running the backtest.**   Otherwise it's too easy to retroactively justify whatever the   numbers show. - **Always check for lookahead bias.** If your backtest peeks at   data that wasn't available at decision time, you're testing a   fantasy — not a strategy. - **Always paper-trade before going live.** Backtest stats lie. Paper   exposes the gap. - **Walk-forward validation is mandatory for any rule you'd deploy.**   In-sample fit is easy. Held-out positive P&L is the bar. The   agentic-decode loop enforces this by default; if you wire a custom   decoder, do the same. - **Negative results are data.** A strategy that performed worse than   hoped is still a unit of learning. Save the result, document why   you expected better, and move on. - **Never tune to a single backtest result.** If you tweak parameters   until one backtest looks great, you've overfit. Use the evolutionary   loop (`wallet-evolve.py`) or the agentic loop (`agentic-decode.py`)   to search broadly, not narrowly.  ## When to touch files in this folder  `lab/` files are FRAMEWORK + RUNNABLE research code. Editing them affects backtest results and paper-trade behavior, but NOT live trading directly. Safe to iterate quickly.  The exception: anything in `bear-scout/runners/strategy-registry.json` with `status: live` is being read by the live runner. Changing it affects live behavior. Treat live entries with the same care as edits to `bots/src/strategies/`.  ## Event-driven achievements  Several lab actions automatically trigger achievements (defined in [`../achievements/definitions.json`](../achievements/definitions.json) and tracked by [`../src/pbx_trader_lab/achievements.py`](../src/pbx_trader_lab/achievements.py)):  - `wallet_decoded` — `wallet-evolve.py` produces `evolution.json` - `first_backtest` — any runner produces a `backtest_completed` event - `sharpe_5` — backtest result with Sharpe > 5 - `sharpe_20` — backtest result with Sharpe > 20 - `ten_thousand_tests` — cumulative backtest count ≥ 10,000  Run `./pbx achievements` from the repo root to see what's unlocked.
