# bear-scout/runners — Research runner scripts  All runnable research code lives here. The naming convention makes the folder grep-able and skim-able.  ## Naming convention  | Pattern | What it is | |---------|-----------| | `paper-trade.py` | The paper trader (long-lived, supervised by pm2) | | `backtest-<thing>.py` | One-off backtest scripts (`backtest-velocity.py`, `backtest-percity.py`) | | `<thing>-sweep.py` | Parameter sweep scripts that try multiple values | | `<thing>-analyze.py` | Analysis scripts that produce reports from existing data | | `_<thing>.py` | Helper modules + internal scratch (underscore prefix = "not a top-level runner, just a helper or a one-off check") | | `_*_log.txt` | Captured output from past runs (gitignored by default) |  The `_` prefix is the most useful convention here: it lets you scan the folder and immediately see which scripts are entry points vs which are support code. If you have to grep for "what runner does X", start with the no-prefix files.  ## strategy-registry.json — the canonical strategy list  Every strategy (paper or live) has an entry in `strategy-registry.json`. The shape of an entry:  ```json {   "id": "<your-strategy-id>",   "status": "paper",   "description": "What this strategy does, in one sentence",   "entry": {     "filters": [       { "kind": "...", "params": { ... } }     ]   },   "dca": {     "max_legs": 0,     "trigger_drawdown_pct": 0   },   "exit": {     "kind": "...",     "params": { ... }   },   "stats": {     "backtest_window": "<date> to <date>",     "win_rate_pct": null,     "avg_pnl_pct": null,     "max_drawdown_pct": null   } } ```  `status` is one of: - `paper` — runs in the paper trader, no real money - `live` — runs against real funds (gated by consent rules — see   the project's behavior-rules doc) - `archived` — kept for reference, not run  The framework doesn't ship with tuned strategies — what's in `strategy-registry.json` is a template entry showing the format. Building your own strategies is the point of the roadmap (Sections 3 and 4).  ## Output conventions  Runners write outputs to `~/.pbx-lab/` rather than into the repo:  | Output | Where | |--------|-------| | Paper trade ticks | `~/.pbx-lab/paper-trades.jsonl` | | Paper trade heartbeat | `~/.pbx-lab/paper-trade-heartbeat` | | AQI cache | `~/.pbx-lab/aqi-snapshot.json` | | Daily digest | `~/.pbx-bear-scout/data/daily-digest-<date>.md` | | Alerts | `~/.pbx-lab/alerts.jsonl` | | Backtest reports | `~/.pbx-lab/backtests/<runner>/<timestamp>/` |  Anything in `~/.pbx-lab/` survives a repo wipe and is meant to be treated as runtime state. Don't commit it.
